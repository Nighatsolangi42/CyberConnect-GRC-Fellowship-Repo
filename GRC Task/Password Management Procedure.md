**1. Purpose**
To define the step-by-step process for creating, managing, and securing passwords in Cyber Connect.

**2. Scope**
Applicable to all IT systems and employees within the IT department.

**3. Roles & Responsibilities**

CEO

1) Approves procedure and ensures enforcement.

IT Department

1) Implements password controls. 
2) Handles password resets and monitoring. 
3) Ensures compliance with policy. 

Employees

1) Follow password requirements. 
2) Maintain confidentiality of credentials. 
3) Report security incidents. 

**4. Procedure**

**4.1 Password Creation**
1.	Employee creates a password meeting defined complexity requirement. 
2.	IT system enforces minimum length and character rules. 
3.	Default passwords are changed at first login.
   
**4.2 Password Usage**
1.	Employees must use unique passwords for each system. 
2.	Password sharing is strictly prohibited. 
3.	Passwords must not be written or stored insecurely.
   
**4.3 Password Storage**
1.	IT ensures passwords are encrypted or hashed in systems. 
2.	Approved password managers may be used where applicable.
   
**4.4 Password Reset Process**
1.	Employee requests reset through IT. 
2.	IT verifies identity of the employee. 
3.	Temporary password is issued. 
4.	Employee must change password upon next login. 

**4.5 Password Change**
1.	System prompts password change every 60–90 days. 
2.	Immediate change required if compromise is suspected. 

**4.6 Account Security**
1.	IT enables MFA where possible. 
2.	Accounts are locked after multiple failed attempts. 
3.	IT investigates suspicious login activity. 

**4.7 Incident Reporting**
1.	Employees report suspected password compromise to IT immediately. 
2.	IT investigates and enforces password reset. 
3.	Actions are logged for audit purposes. 

**5. Compliance**
•	All users must comply with this procedure. 
•	Violations may result in disciplinary action. 

**6. Review**
•	Reviewed annually or upon significant changes.

**Cyber Connect: Password Management Policy (Dummy)**

**1. Purpose**
To ensure secure creation, usage, and management of passwords for protecting organizational systems and data.

**2. Scope**
Applies to all IT systems, applications, and users within the IT department of Cyber Connect.

**3. Roles & Responsibilities**
CEO:
- Approves the policy.
- Ensures organizational support for security practices.

IT Department:
- Enforces password standards and controls.
- Manages password systems, resets, and monitoring.
- Ensures secure storage (e.g., hashing, encryption).

IT Employees:
- Create and maintain secure passwords.
- Do not share or reuse passwords.
- Report any suspected compromise immediately.
  
**4. Policy Requirements**
4.1 Password Creation:
- Minimum length: 8–12 characters.
- Must include uppercase, lowercase, numbers, and symbols.
- Avoid common or easily guessable passwords.

4.2 Password Usage:
- Passwords must be unique per system.
- Sharing passwords is strictly prohibited.
- Default passwords must be changed immediately.

4.3 Password Storage:
- Passwords must not be stored in plain text.
- Use approved password managers where applicable.

4.4 Password Change:
- Change passwords every 60–90 days.
- Immediate change required if compromise is suspected.

4.5 Account Security:
- Enable Multi-Factor Authentication (MFA) where possible.
- Accounts must lock after multiple failed login attempts.
  
**5. Compliance & Enforcement**
Non-compliance may result in disciplinary action. IT will monitor adherence and enforce controls.

**6. Review**
Policy to be reviewed annually or as required.
